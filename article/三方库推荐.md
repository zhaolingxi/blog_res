虽然C++er最喜欢和擅长的就是造轮子，但是在交付任务的时候，在工期和产品奇怪要求的拷打下，我们最后还是会回到有人不断维护和更新的三方库中。

根据这几年的使用体验，记录和推荐以下三方库，推荐的标准是**简单易用、性能不差、开放源码、有人维护、允许一定程度的商用**，同时每个方面仅推荐一个库。

## 日志类
1.基础日志库
plog：轻量、易用的 C++ 日志库（header-only）,默认输入与cout兼容，支持自定义日志格式，支持分级日志、滚动文件、线程安全；接口简洁，适合嵌入到各类项目。
[plog](https://github.com/SergiusTheBest/plog.git)

2.错误追踪类
backward-cpp：在崩溃或异常时生成可读的堆栈回溯信息，支持符号解析与源码位置定位，便于快速问题排查。
[backward-cpp](https://github.com/bombela/backward-cpp.git)

## 多任务并发相关
1.单机多任务并发封装
asio：老牌跨平台异步 I/O 库，提供定时器、套接字、串口等事件驱动封装；推荐独立源码编译使用。
[asio](https://github.com/chriskohlhoff/asio.git)

2.简易回调触发
eventpp：现代 C++ 事件/回调管理库，支持事件分发、监听器注册与过滤器，API 简洁、易扩展。
[eventpp](https://github.com/wqking/eventpp.git)

## 网络相关
1.网络库
crow：C++ 的微型 Web 框架，类似 Flask 的路由风格，内置 HTTP/JSON/WebSocket，基于 Asio，适合快速搭建服务。
[crow](https://github.com/ipkn/crow.git)

## 数据库
1.单机数据库
sqlite3：零配置、嵌入式关系型数据库，单文件部署、性能稳定，广泛用于本地存储和嵌入式场景。
[sqlite3](https://github.com/mattn/go-sqlite3.git)

2.跨设备
mysql：主流开源关系型数据库，生态完善、跨平台；C/C++ 连接器可用，适合跨设备与服务端数据管理。(c++er推荐直接用发行包或者docker)
[mysql](https://github.com/mysqljs/mysql.git)

## 消息中间件
1.本机内
fastdds：遵循 DDS 标准的发布/订阅通信中间件，低时延、实时性好，适合机器人与车载分布式系统。
[fastdds](https://github.com/eProsima/Fast-DDS.git)

2.集群中
kafka：经典的高吞吐分布式日志与流处理平台，支持分区、副本与持久化，常用于微服务解耦与数据管道。
[kafka](https://github.com/apache/kafka.git)
[cppkafka](https://github.com/mfontanini/cppkafka.git)

3.物联网
emqx：高性能 MQTT 5.0 消息服务器，支持集群与设备海量连接，优秀的网络适应能力，适合物联网场景。
[emqx](https://github.com/emqx/emqx.git)

## 几何相关：
1.碰撞检测
bullet3：成熟的物理与碰撞检测引擎，支持刚体、约束与宽相窄相碰撞，仿真与游戏领域广泛使用。
[bullet3](https://github.com/bulletphysics/bullet3.git)

2.模型加载
assimp：通用 3D 模型导入库，支持大量文件格式（FBX/OBJ/GLTF 等），提供统一的场景/网格接口。
[assimp](https://github.com/assimp/assimp.git)

## 压缩与解压
1.通用文件
zlib：经典的通用压缩库，提供 Deflate/Gzip 等算法，稳定可靠、跨平台。
[zlib](https://github.com/madler/zlib.git)

2.消息类
protobuf：高效的二进制序列化协议，跨语言/跨平台，支持向前/向后兼容与 schema 演进。虽然使用体验一般，但是已经是某种意义上的通行标准。
[protobuf](https://github.com/protocolbuffers/protobuf.git)

## UI类
1.简易UI
imgui：即时模式 GUI 库，零样式负担、开发效率高，常用于调试面板、工具与内嵌编辑器。
[imgui](https://github.com/ocornut/imgui.git)

2.复杂UI
qt：成熟的跨平台应用框架，提供窗口控件、QML/图形/网络/多媒体等完整组件，适合复杂桌面与嵌入式 UI。（规模化商用请自行斟酌付费）
[qt](https://www.qt.io/)

## 文本处理
1.json
jsoncpp：简洁的 C++ JSON 解析/生成库，易集成、API 友好，适合通用配置与数据交换。
[jsoncpp](https://github.com/open-source-parsers/jsoncpp.git)

2.yaml
yaml-cpp：C++ 的 YAML 解析与生成库，支持节点遍历、序列化，常用于复杂配置文件。
[yaml-cpp](https://github.com/jbeder/yaml-cpp.git)

## 算法类
1.通用数学库
gsl：通用数值计算库，涵盖矩阵/统计/优化/积分等；C 接口为主，C++ 可调用，功能全面。
[gsl](https://github.com/ampl/gsl.git)

2.约束求解库
or-tools：运筹优化工具包，内置 CP-SAT、线/整数规划与车辆路径等，发布版本兼容SCIP、cbc、glop以及付费商用求解器，适合调度与约束求解，一站式解决方案。
[or-tools](https://github.com/google/or-tools.git)

3.矩阵运算库
Eigen：头文件分发的高性能线性代数库，提供矩阵/向量、分解与数值求解，表达式模板优化良好。
[Eigen](https://libeigen.gitlab.io/)

## 专业领域
1.CAD领域
OCCT: 工业级几何内核与 CAD 框架，提供 B-Rep/曲面（含 NURBS）建模、布尔/倒角/偏置等几何运算，拓扑结构管理（TopoDS），形状修复与网格化，含可视化（AIS/V3d）与数据交换模块（如 STEP/IGES），适合从几何算法到完整 CAD 应用的二次开发。
[OCCT](https://github.com/Open-Cascade-SAS/OCCT)

2.高精地图
libopendrive：面向 OpenDRIVE 高精地图格式的解析/读取库，支持道路几何、连接与拓扑建模，常用于仿真与自动驾驶地图处理。
[libopendrive](https://github.com/pageldev/libOpenDRIVE.git)

3.智驾场景仿真
esmini: 面向 OpenSCENARIO 的轻量级场景播放器/引擎，内置 OpenDRIVE 路网支持，可脚本化运行场景、复现交通参与者行为，支持可视化与无头模式、可嵌入到 SIL/HIL 流程中，适合用作标准场景的快速回放与回归测试。
[esmini](https://github.com/esmini/esmini.git)

4.智驾综合仿真
carla: 基于 Unreal Engine 的开源自动驾驶仿真平台，提供城市级地图、天气与光照、交通流与行人为一体的逼真环境，内置丰富传感器（相机/深度/分割/LiDAR/Radar 等）与 Python API、Traffic Manager、ROS 桥接，适合端到端感知—决策—控制算法的开发与评测。
[carla](https://github.com/carla-simulator/carla.git)
